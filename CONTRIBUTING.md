# Contributing Guidelines

**System Classification**: A.3 He-II λ-Transition κ Analysis  
**Author**: Oleksii Onasenko  
**Developer**: SubstanceNet

## Overview

Contributions to this project are welcome. This document outlines the process for contributing.

## Scope

This project analyzes the λ-transition of liquid helium-4 using the emergence parameter κ framework. Contributions should maintain scientific rigor and align with this scope.

## How to Contribute

### Reporting Issues

1. Check existing issues to avoid duplicates
2. Provide clear description of the problem
3. Include steps to reproduce (if applicable)
4. Specify environment details (OS, Python version)

### Proposing Changes

1. Fork the repository
2. Create a feature branch
3. Make changes with clear commit messages
4. Ensure code passes existing tests
5. Submit pull request with detailed description

## Code Standards

### Python Code
- Follow PEP 8 style guidelines
- Include docstrings for all functions and classes
- Maintain existing code structure
- Add comments for complex algorithms

### Documentation
- Use academic English
- Provide clear explanations
- Include references where appropriate
- Maintain consistent formatting

## Testing

- Verify that analysis pipeline runs without errors
- Check that results match expected values (κ ≈ 1.00)
- Ensure figures generate correctly

## Review Process

All contributions will be reviewed for:
- Scientific accuracy
- Code quality
- Documentation clarity
- Consistency with project standards

## Attribution

Significant contributions will be acknowledged in project documentation.
